# Contributing to Eriska

Thank you for your interest in contributing to Eriska! This document provides guidelines to help you through the contribution process.

## Contribution Workflow

1. **Fork**: Fork the repository to your own GitHub account.
2. **Branch**: Create a new branch for each feature or bug fix.
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Commit**: Commit your changes with clear, descriptive messages.
   ```bash
   git commit -m "Add some amazing feature"
   ```
4. **Push**: Push your changes to your fork.
   ```bash
   git push origin feature/amazing-feature
   ```
5. **Pull Request**: Submit a Pull Request to the `main` branch of the original repository.

## Bug Reports

If you encounter an issue, please open a new Issue and include:
- Steps to reproduce the error.
- Expected behavior.
- Observed behavior.
- Screenshots (if applicable).

## Coding Standards

- Please follow the existing coding style.
- For Python, follow PEP 8.
- For JavaScript/TypeScript, use ESLint and Prettier.

Thank you for helping make the IoT world safer!
